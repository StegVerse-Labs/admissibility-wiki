# Transition Table

The Transition Table is a StegVerse formalism surface for classifying proposed state changes and their governance posture.

## Visual Reference

The public Site bridge for the full transition-table visual/reference page is:

```text
https://stegverse-labs.github.io/Site/admissibility-wiki.html
```

This wiki page should be treated as the terminology and formalism mirror. The Site page is the current public visual/presentation surface until a dedicated transition-table visual artifact is mirrored into this wiki.

## Purpose

The table should help make transition review explicit.

It can represent:

- transition type;
- authority class;
- policy reference;
- evidence posture;
- review posture;
- admissibility result;
- commit-time validity;
- receipt status.

## Whole-Table Relationship

The full table is intended to show that an action should not be treated as governed merely because it is visible, animated, approved earlier, or technically executable.

A transition should be evaluated through at least these dimensions:

| Dimension | Question |
|---|---|
| Transition | What state change is proposed? |
| Actor | Who or what is attempting the change? |
| Authority class | What standing does the actor claim? |
| Policy reference | Which rule governs this transition? |
| Evidence posture | What evidence exists, and is it current? |
| Review posture | What review has occurred? |
| Drift posture | Has any relevant state changed before commit? |
| Decision result | Is the transition allowed, denied, escalated, refused, deferred, or superseded? |
| Commit-time validity | Did standing still exist at the moment consequence would bind? |
| Receipt | Can the decision path be reconstructed? |

## Why It Matters

The table makes it harder to confuse animation, visibility, approval, or execution with governance.

A visible transition is not automatically an admissible transition.
