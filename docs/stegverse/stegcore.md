# StegCore

StegCore is the StegVerse core layer for governed transition logic.

Within the Admissibility Wiki, StegCore should be treated as an implementation surface, not as the definition of admissibility itself.

## Role

StegCore may host or expose:

- transition checks;
- receipt generation;
- policy reference handling;
- authority class mapping;
- refusal and escalation logic;
- replay or verification utilities.

## Boundary

StegCore should not be described as proving all governance claims by itself.

Its claims depend on the policies, evidence sources, authority providers, and enforcement managers it is connected to.
