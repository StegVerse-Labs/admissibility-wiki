# Transition Origin Governance

## Purpose

Every native StegVerse term, concept, action, state, consequence, receipt, authority, review posture, evidence posture, governance artifact, and execution artifact should identify its origin within the Transition Table and/or Transition Blocks.

The purpose of this requirement is to prevent terminology drift and preserve reconstructability of meaning.

## Core Rule

Definitions are not authoritative merely because they exist in the wiki.

Definitions derive standing from the Transition Table, Transition Blocks, or formally accepted relationships.

## Required Section

All glossary pages should eventually contain a section named:

```text
Transition Origin
```

## Minimum Structure

```text
Transition Table Elements:
- <element>

Transition Blocks:
- <block>

Derived From:
- Transition Table
- Transition Blocks
- Both

Standing:
- Native StegVerse Concept
- Derived StegVerse Concept
- External Equivalent
- External Overlap
- External Adjacent

Authority Source:
- <source>

Commit-Time Relevance:
- Yes/No

Reconstructability Relevance:
- Yes/No
```

## Orphan Rule

A term without a recorded transition origin should be treated as unclassified until reviewed.

Unclassified does not imply invalid.

Unclassified means origin standing has not yet been documented.

## External Mapping Rule

External terms may map to a StegVerse term.

External mappings do not modify the native transition origin.

## Future Validation Goal

Future validators should identify:

- glossary terms without transition origin sections;
- transition origins referencing unknown elements;
- transition origins referencing unknown blocks;
- inconsistent standing classifications;
- ontology records missing transition-origin metadata.
