This page explains how the StegVerse repository standards surface relates to admissibility.

## Source Authority

StegVerse-Labs/repo-standards remains authoritative for repository standards, schemas, templates, validators, correction tooling, and release readiness.

This wiki is explanatory. It does not validate source standards, grant correction authority, or convert a release candidate into a final release.

## Admissibility Position

A repository standard is admissible only when a repository can prove conformance from mapped evidence.

A claim of conformance is not enough.

Required evidence may include:

- standards registry
- metadata declaration
- README source map
- validator output
- correction plan
- receipts or reports
- handoff source of truth

## Standards Admissibility Result Classes

- ALLOW: required evidence passed validation.
- DENY: required evidence failed validation.
- FAIL-CLOSED: evidence is missing, stale, contradictory, or unverifiable.
