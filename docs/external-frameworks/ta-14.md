## Publicly declared architecture

TA-14 publicly describes an eight-stage consequence-bearing route:

```text
Reality
-> Record
-> Continuity
-> Admissibility
-> Binding
-> Commit
-> Execution
-> Outcome
```

Its public materials describe the architecture as proof-bound execution governance, require admissibility before consequence, distinguish permission from admissibility, require continuity and evidence integrity, and describe the execution boundary as fail-closed when required proof is absent.

## Open parentage dispute

TA-14 publicly claims parent-architecture status. Current public review confirms that the claim is explicit and repeated. It does not independently establish that other architectures are descendants, contained components, or derivative implementations.

The reciprocal test must therefore examine whether TA-14 can reconstruct an external framework accurately without presupposing that framework's subordination to TA-14.
