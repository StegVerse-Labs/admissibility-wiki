# Model Context Protocol

## Evidence posture
`sourced_intake` — canonical protocol specification captured; no tool-execution observation claimed.

## Published scope
MCP defines a client-server protocol for exposing context, resources, prompts, and tools to AI applications, including capability negotiation and authorization guidance.

Canonical source: https://modelcontextprotocol.io/specification/2025-06-18

## StegVerse relationship
`adjacent`

MCP can carry tool and resource contracts into an interoperability path. Capability discovery and invocation preparation remain distinct from StegVerse consequence-binding authority.

```text
tool discovered != tool authorized for this transition
capability advertised != current delegation
call prepared != commit admissible
```

## Benchmark relevance
`preparation_boundary`, `commitment_boundary`, `authority_boundary`, `interoperability_path`

## Non-claims
Protocol interoperability does not establish transition admissibility, current standing, or execution authority.
