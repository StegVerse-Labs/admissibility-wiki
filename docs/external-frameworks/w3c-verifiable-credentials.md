# W3C Verifiable Credentials

## Evidence posture
`sourced_intake` — W3C Recommendation captured; no StegVerse interoperability observation claimed.

## Published scope
The Verifiable Credentials Data Model 2.0 defines a model for credentials and presentations involving issuer, holder, verifier, credential status, and securing mechanisms.

Canonical source: https://www.w3.org/TR/vc-data-model-2.0/

## StegVerse relationship
`adjacent`

A verified credential may become identity, claim, or status evidence. The verifier's trust decision and the transition's current authority remain distinct.

## Benchmark relevance
`authority_boundary`, `evidence_freshness_boundary`, `reconstruction_boundary`, `interoperability_path`

## Non-claims
Credential verification is not execution authority. Issuer trust is not inherited automatically. Inclusion does not establish StegVerse standing.
