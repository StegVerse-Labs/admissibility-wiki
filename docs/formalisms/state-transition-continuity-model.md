# State Transition Continuity Model

## Status

Canonical source repository: `Admissible-Existence/STCM`

Wiki state: watching

Detail status: initial public-safe canonical detail page

## Definition

State Transition Continuity Model is a canonical formalism for evaluating whether continuity is preserved across state changes.

It supports review of whether a transition remains connected to its prior authority, evidence, policy, boundary, and receipt context.

## Source Boundary

Admissible-Existence defines the formalism. Publisher publishes papers and public exposition. The Admissibility Wiki mirrors, relates, crosswalks, and discovers relationships.

The wiki is not the source authority for this formalism.

## Scope

State Transition Continuity Model applies where a system must determine whether a later state can still be connected to the prior state that gave a transition standing.

## Purpose

The purpose of State Transition Continuity Model is to prevent systems from treating a state change as harmless merely because a prior review artifact or prior approval remains visible.

## Core Constructs

| Construct | Role |
|---|---|
| State | A recorded system, policy, authority, evidence, or boundary condition at a moment. |
| Transition | A change from one state to another. |
| Continuity | Whether the transition path preserves the needed relationship between states. |
| Discontinuity | A break that prevents prior standing from carrying forward. |
| Rebinding | A renewed check against current state. |
| Receipt | Record supporting reconstruction of continuity or discontinuity. |
