## Definition

Admissible-Existence Validation Factory is a canonical validation-infrastructure formalism for generating, organizing, or applying tests that evaluate admissibility-related artifacts, registries, receipts, and transition records.

It supports inspection of whether a formalism has source-confirmed validation candidates, executable tests, or reproducible examples.

## Source Boundary

Admissible-Existence defines the formalism or validation-infrastructure boundary. Publisher publishes papers and public exposition. The Admissibility Wiki mirrors, relates, crosswalks, and discovers relationships.

The wiki is not the source authority for this validation factory.

## Scope

Admissible-Existence Validation Factory applies where formalism claims, transition examples, receipts, registries, or runtime outcomes need source-confirmed validation surfaces.

## Purpose

The purpose of Admissible-Existence Validation Factory is to prevent validation language from remaining informal when a formalism needs explicit tests, fixtures, expected outcomes, or reproducible proof candidates.

## Core Constructs

| Construct | Role |
|---|---|
| Validation target | Artifact, formalism, receipt, registry, or transition outcome being tested. |
| Test fixture | Input data or state used to evaluate a validation target. |
| Expected outcome | Result that a valid implementation or example should produce. |
| Validation receipt | Record showing what was checked and what result occurred. |
| Proof candidate | Source-confirmed artifact that may support a formal proof path. |
| Reference implementation | Source-confirmed executable implementation or example. |
