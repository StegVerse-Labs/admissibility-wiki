# Transition Table

## Status

Canonical source repository: `Admissible-Existence/TT`

Wiki state: watching

Detail status: initial public-safe canonical detail page

## Definition

Transition Table is a canonical formalism for classifying transition posture.

It helps separate what happened, what was reviewed, what had authority, and what can bind consequence.

## Source Boundary

Admissible-Existence defines the formalism. Publisher publishes papers and public exposition. The Admissibility Wiki mirrors, relates, crosswalks, and discovers relationships.

The wiki is not the source authority for this formalism.

## Scope

Transition Table applies where a system needs to inspect a transition across input, review, authority, evidence, policy, boundary, commitment, receipt, and outcome.

## Purpose

The purpose of Transition Table is to prevent a single label such as approved, visible, or executed from hiding the actual governance posture of a transition.

## Core Constructs

| Construct | Role |
|---|---|
| Transition | Proposed or attempted state change. |
| Transition posture | Current classification of the transition. |
| Authority class | Kind of authority claimed or required. |
| Policy reference | Governing rule or constraint. |
| Evidence posture | Evidence state supporting or failing to support the transition. |
| Review posture | Review state and standing. |
| Commit-time validity | Whether standing remains at the moment consequence would bind. |
| Receipt | Record supporting reconstruction of the transition path. |
