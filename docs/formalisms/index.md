# Formalisms

## Purpose

The Admissibility Wiki should include each formalism that helps companies, researchers, public institutions, independent builders, and individual contributors converge on governed AI adoption faster.

This section is a public mirror and convergence surface. It does not replace the originating repositories, formal proof artifacts, executable tests, decision records, or source authority.

## Current Formalism Scope

A formalism page may document:

- the formalism name;
- source repository or source artifact;
- purpose;
- transition-table relationship;
- authority boundary;
- evidence posture;
- replay or reconstruction path;
- machine-readable artifact links;
- proof/test status;
- adoption status;
- non-claims.

## Formalism Record States

| State | Meaning |
|---|---|
| intake | A formalism has been identified but not mapped. |
| mirrored | A public wiki page exists and links to source artifacts. |
| crosswalked | The formalism has mapped glossary, ontology, receipt, and proof-path relationships. |
| validated | The formalism has a validator, example, replay path, or formalism-test artifact. |
| superseded | A newer formalism has replaced or narrowed this one. |

## Authority Boundary

The wiki may explain or mirror a formalism, but it must not overclaim what the formalism proves.

```text
Wiki page = public explanation and convergence layer
Source repository = originating artifact authority
Formalism test = executable proof or validation authority when present
Decision record = accepted governance disposition
Continuity receipt = reconstructability and handoff evidence
StegCore = admissibility interpretation where applicable
```

## Contribution Rule

External companies, researchers, or contributors may propose formalism pages, but each proposal should identify:

```text
source artifact
claim boundary
relationship to existing wiki terms
evidence posture
validation posture
non-claims
```

## Next Safe Build Target

Create a machine-readable formalism registry and validator so formalism pages can be added without duplicate or unsupported claims.
