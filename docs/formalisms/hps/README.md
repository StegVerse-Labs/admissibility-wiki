# Harmonic Principle of Standing

## Purpose

The Harmonic Principle of Standing (HPS) explains how governed systems restore standing over time.

The canonical HPS formalism is maintained in:

```text
Admissible-Existence/HPS
```

This admissibility-wiki page is a public explanatory mirror. It does not replace the canonical formalism, schema, verifier, or test fixtures.

## Core statement

```text
A governed ecosystem behaves as a harmonic oscillator.

Transition pressure displaces standing.
Governance provides restoring force.
Damping prevents unstable oscillation.
Independent entities synchronize through governed resonance.
The heartbeat is the observable restoration of standing.
Receipts preserve phase history.
Master Records preserves continuity.
```

## Why HPS belongs in admissibility

HPS preserves a core admissibility distinction:

```text
Integrity is not standing.
Standing is not admissibility.
Admissibility is not execution authority.
```

A system may be online and active while standing is degraded or failed. HPS describes how that condition should be observed, measured, and routed.

## Capability windows

HPS defines when governed capabilities become dynamically available and when they expire.

```text
A capability becomes available when the heartbeat restores the standing required for that transition class.
A capability expires when phase, authority, evidence, policy, coordinate, or reconstruction no longer supports it.
No capability may borrow standing from a prior heartbeat after its window has closed.
```

## Standing equation

The standing equation is summarized as:

```text
S(t) = B(t) + C(t) + A(t) + R(t) - D(t)
```

Where:

```text
B(t) = body / integrity support
C(t) = certificate / provenance support
A(t) = coordinate / ordering support
R(t) = reconstruction support
D(t) = drift, decay, stale evidence, authority failure, or continuity break
```
