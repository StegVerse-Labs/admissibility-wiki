# Triad Governance Model

## Status

Canonical source repository: `Admissible-Existence/Triad`

Wiki state: watching

Detail status: initial public-safe canonical detail page

## Definition

Triad Governance Model is a canonical formalism for evaluating governance through a multi-role structure rather than a single actor, reviewer, or automated authority.

It supports inspection of whether initiation, arbitration, observation, review, or escalation roles are separated enough to preserve governance standing.

## Source Boundary

Admissible-Existence defines the formalism. Publisher publishes papers and public exposition. The Admissibility Wiki mirrors, relates, crosswalks, and discovers relationships.

The wiki is not the source authority for this formalism.

## Scope

Triad Governance Model applies where a transition, decision, publication, or execution attempt requires multiple governance roles, quorum structure, or role-separated review before consequence can bind.

## Purpose

The purpose of Triad Governance Model is to prevent governance from collapsing into a single unobserved or self-authorizing path when the transition requires distributed standing.

## Core Constructs

| Construct | Role |
|---|---|
| Initiator | Role that proposes or begins a transition. |
| Arbiter | Role that resolves or adjudicates contested standing. |
| Observer | Role that preserves visibility, receipt context, or independent reconstruction. |
| Quorum | Required role participation or threshold. |
| Role separation | Boundary preventing one role from silently substituting for another. |
| Receipt | Record supporting reconstruction of role participation and standing. |
