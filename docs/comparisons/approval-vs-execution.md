# Approval vs Execution

Approval is a prior permission signal.

Execution is the act of carrying out a transition.

Neither one alone proves admissibility.

## Core Distinction

A transition may have been approved earlier but become inadmissible before execution.

A transition may also be technically executable while lacking commit-time authority.

## Practical Rule

Before execution binds, the system should check whether the authority, policy, evidence, and context state still support the consequence.
