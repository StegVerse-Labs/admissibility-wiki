# Auditability vs Admissibility

Auditability asks whether a system can later inspect what happened.

Admissibility asks whether a transition had standing when it bound.

## Core Distinction

A system can be auditable after the fact but still allow inadmissible transitions at commit time.

Auditability is important, but it is not a substitute for commit-time governance.

## Practical Rule

Audit should preserve evidence.

Admissibility should constrain consequence before it binds.
