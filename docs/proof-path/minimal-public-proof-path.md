# Minimal Public Proof Path

The minimal public proof path is the smallest public demonstration that can show how StegVerse-style admissibility works without requiring a reader to understand the entire ecosystem.

## Required Artifacts

A minimal proof path should include:

1. one transition cell example;
2. one transition receipt;
3. one replay or verification command;
4. one refusal or rejection case;
5. one explanation of authority class, policy reference, admissibility result, and commit-time validity.

## Success Criteria

A reviewer should be able to answer:

- what transition was proposed;
- what authority class was claimed;
- what policy reference applied;
- what evidence was used;
- what result occurred;
- whether the transition was allowed, refused, rejected, or escalated;
- whether the receipt can be replayed or checked.
