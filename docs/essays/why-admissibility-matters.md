# Why Admissibility Matters

Modern systems often focus on whether an action can be performed, whether it was approved, or whether it can be audited later.

Admissibility adds a different question:

> Should this transition be allowed to bind under the authority, policy, evidence, and context state that exists now?

## The Gap

A system can be technically capable of execution while its permission basis has drifted.

A system can preserve logs while failing to prevent an inadmissible consequence.

A system can show movement while failing to show standing.

## The Governance Claim

Admissibility matters because consequence should not bind merely because execution is possible.

Consequence should bind only when the governing basis remains reconstructable, current, and sufficient at commit time.
